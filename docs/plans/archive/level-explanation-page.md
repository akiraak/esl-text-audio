# レベルについての説明ページを作る

## 目的・背景

TODO.md に記載済みのタスク。現状 [server.js](../../server.js) のビューアでは、一覧・詳細ページに `config.level`（例: `A2`）をそのまま表示するのみで、
CEFRレベルが何を意味するか（総語数目安・文長・語彙範囲など）を確認する手段がない。
レベル定義自体は [docs/specs/esl-level-spec.md](../specs/esl-level-spec.md) に一元化されているため、この内容をビューア上の説明ページとして閲覧できるようにする。

## 対応方針

- `server.js` に `GET /levels` ルートを追加し、`docs/specs/esl-level-spec.md` の「CEFR レベル定義」表をそのまま読み込んで表示する
  - スペックファイルをサーバ起動時の唯一の情報源とし、内容の二重管理をしない（Markdownをそのまま `marked` でレンダリングする）
  - ジャンル別適性表・事実チェック方針は今回のスコープ外（レベル説明に閉じる）。表示するのは「CEFR レベル定義」セクションのみとし、他セクションは対象外にする
- 一覧ページ（`GET /`）のヘッダー付近に `/levels` へのリンクを追加し、迷わず辿り着けるようにする
- 既存の `layout()` を使い、見た目のトーンを既存ページに揃える

## 影響範囲

- `server.js` の変更のみ（新規ファイルなし）

## テスト方針

- 自動テストは作らない（既存の表示専用サーバの方針を踏襲）
- 実装後、サーバを起動し `curl` で `/levels` の表示内容、および `/` からのリンク遷移を実地確認する

- [ ] Phase 1: `server.js` に `/levels` ルート追加（esl-level-spec.md の「CEFR レベル定義」セクション抽出・表示）、一覧ページへのリンク追加
- [ ] Phase 2: 動作確認（`/levels` 表示、一覧ページからのリンク遷移）
