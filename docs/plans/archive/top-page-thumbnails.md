# トップページの記事一覧にサムネを表示

## 目的・背景

TODO.md 記載のタスク。トップページ（`renderIndex`）の記事一覧は現状テキスト情報のみで、
イラスト生成済みのトピックでもサムネイルが表示されず単調。`lib/site.js` には既に
トピック単位のイラストを扱うヘルパー（`latestIllustrationVersion`, `illustrationBlock`）があるため、
これを流用してサムネイル表示を追加する。

## 対応方針

- `lib/site.js` の `renderIndex` 内、各 `<li>` 生成時に `latestIllustrationVersion(id)` でイラストの有無を確認し、
  存在すれば `<img>` サムネイルを `<li>` 内に追加する（`illustrationBlock` は本文用の大きい表示なので流用せず、
  サムネイル用に小さいサイズで直接 `href()` を使って組み立てる）
- イラストが無いトピックは従来通りテキストのみの表示のまま（レイアウト崩れしないよう、サムネイル有無に関わらず
  `<li>` は横並び flex レイアウトにし、画像枠はプレースホルダーなしで省略する）
- `layout()` の `<style>` に `.text-list li` を flex 化するCSS・`.thumb` のサイズ・角丸・object-fit を追加

## 影響範囲

- `lib/site.js` の `renderIndex` と `layout()` の `<style>` のみ。動的サーバ・静的ビルド両方に反映される

## テスト方針

- `npm start` でローカルサーバを起動し、イラスト生成済み・未生成の両トピックが混在する状態でトップページを目視確認
  - サムネイルが正しい画像を指しているか（クリックで記事に飛ぶか）
  - イラスト未生成のトピックでレイアウトが崩れないか
  - モバイル幅でも崩れないか
