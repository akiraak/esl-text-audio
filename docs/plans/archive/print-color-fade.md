# 印刷時に文字の色が薄くなる問題の調査と対応

## 背景・目的

TODO.md に記載の「印刷時に文字の色が薄くなる原因の調査を対応策を考える」に対応する。
`lib/site.js` が組み立てる記事ページ（`server.js` / 静的ビルド共通）を紙に印刷すると、
本文・見出し・メタ情報などの文字が画面表示より薄く見える、という報告に基づく調査。

## 原因調査

`lib/site.js` の `layout()` 内 `<style>` を確認した結果、以下が判明した。

- `@media print` 用のスタイルが一切定義されていない。画面用スタイルがそのまま印刷にも適用される。
- 本文・見出し・メタ情報の文字色がいずれも純黒 (`#000`) ではない、やや薄いトーンの色である。
  - body 本文: `#2f2a3a`（濃い紺紫）
  - 見出し (h1/h2, article h1, .topic, tier-label など): `#3d3560`
  - メタ情報 (.meta, .breadcrumb, .creator-meta, .wc-hint など): `#7a7290`（中間トーンのグレー紫、かなり明るめ）
  - table.variant-matrix td.unavailable / .crumb-sep: `#c9c1d6`（さらに薄いグレー）
- 画面（発光ディスプレイ）+ クリーム色背景 (`#fffaf3`) の組み合わせでは十分なコントラストに見えるが、
  紙面（白背景・反射光）では条件が変わる。
- ブラウザ・OSの印刷パイプラインは既定で `print-color-adjust: economy`（省インク最適化）が有効なことが多く、
  純黒でない色（特に中間トーンの紫グレー）はCMYK変換や省インク処理でさらに薄くレンダリングされやすい。
  背景色・枠線色（`background`, `border`, `box-shadow` など装飾用の薄いオレンジ系色）も
  「背景のグラフィック」を印刷オプションで有効にしない限り印刷されないため、カード状の装飾やバッジは
  枠が消えて見えることもある。
- 上記の複合要因により、「本文が薄く読みにくい」という体感になっていると考えられる。

## 対応方針

`lib/site.js` の `<style>` に `@media print` ブロックを追加し、印刷時のみ以下を上書きする。

1. 本文・見出し・メタ情報などの読み取りに関わる文字色をすべて純黒 (`#000`) 相当に強制する
   （body, h1/h2, article h1, .meta, .topic, .tier-label, .breadcrumb, .creator-meta, badge文字など）。
2. 装飾用の背景色・box-shadow・角丸カードの背景等は印刷では不要なため、白背景・最小限の枠線に簡略化する。
   バッジ (badge, badge-level, badge-tier) は背景色に依存せず判別できるよう、枠線付きにするか
   文字のみ強調する。
3. ナビゲーション目的の要素（`.versions`, `.breadcrumb` のリンク下線など）は印刷時に読む上で
   重要度が低いため、そのままでも問題ないが、リンク色は文字色に統一しておく。
4. 変更は `lib/site.js` の1ファイルのみ（`server.js` と `scripts/build-static-site.js` はこれを共通利用しているため、
   両方に自動的に反映される）。

## 影響範囲

- `lib/site.js` の `<style>` のみ変更。ロジック（HTML組み立て関数）には手を入れない。
- ローカルサーバ (`npm start`) と静的ビルド (`npm run build` → GitHub Pages) の両方に反映される。

## テスト方針

- Puppeteer 等のヘッドレスブラウザ依存はプロジェクトに存在しないため、自動テストでの印刷プレビュー検証は行わない。
- `npm run build` で静的HTMLを生成し、生成された `<style>` に `@media print` ブロックが正しく含まれていることを目視確認する。
- ブラウザで実際のページを開き、印刷プレビュー（Ctrl+P 相当）で文字が黒く表示されることを目視確認する。
