# ペルソナ一覧

本文生成・チェックの各段階で異なる視点を持たせるためのペルソナ定義集。
1ペルソナ1ファイルとし、`workflows/*.md` から該当ファイルを参照して呼び出す。

| # | ペルソナ | 使用ワークフロー | 役割（一言） |
|---|---|---|---|
| 1 | [ESL教材ライター](esl-writer.md) | `outline.md`, `generate.md` | レベル・ジャンルに沿って本文を実際に書く執筆担当 |
| 2 | [学習者シミュレーター](learner-simulator.md) | `generate.md` | 指定レベルの学習者になりきって読み、つまずく箇所を指摘する |
| 3 | [懐疑的ファクトチェッカー](skeptical-fact-checker.md) | `factcheck.md` | 本文の事実主張を疑い、`sources/` と突き合わせる |
| 4 | [簡略化セーフティチェッカー](simplification-safety-checker.md) | `factcheck.md` | ESL向けの易化によって事実の意味が歪んでいないか確認する |
| 5 | [最終エディター](final-editor.md) | `generate.md`（最終）, `brushup.md` | 各ペルソナの指摘を統合し、矛盾する指摘の優先順位を判断して最終稿を確定する |

## ファイル構成のルール

各ペルソナファイルは以下の4セクションで統一する。

- **使用ワークフロー** — どの `workflows/*.md` のどの段階で呼び出すか
- **役割** — 1〜2文でこのペルソナの目的を説明する
- **プロンプト** — 実際にこのペルソナとして振る舞うときの指示文（二人称「あなたは〜」で記述し、そのまま渡して使える形にする）
- **出力形式** — 何を、どういう形式で出力させるか（指摘リスト・OK/NG判定など）

## 運用ルール

- ペルソナを追加・変更する場合は、このファイルの表に反映してから対応する `.md` ファイルを作成・編集する
- ペルソナ同士の指摘が矛盾する場合（例: 学習者シミュレーターの「易しくしてほしい」と懐疑的ファクトチェッカーの「正確さを保ってほしい」）は、[最終エディター](final-editor.md) が優先順位を判断する
- 追加候補（未実装・運用しながら必要になれば追加）:
  - **ストーリー編集者** — 起伏・オチ・意外性など「読んでいて楽しいか」だけを見る編集者視点
  - **CEFR適合性審査官** — `docs/specs/esl-level-spec.md` の表と機械的に照合する、語彙・文長・文法項目の審査役
