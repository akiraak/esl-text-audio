# 記事ページのレイアウト順変更

## 目的・背景

記事ページ（バリアント詳細 `/texts/{topicId}/{variantId}` および記事バージョンページ
`/texts/{topicId}/{variantId}/article/{v}`）の要素の並びを、イラストを先頭に出す形に変更する。

現在の並び:

1. パンくずリンク
2. バージョンリンク
3. タイトル
4. レベル・長さバッジ
5. Author / Created（creator-meta）
6. イラスト
7. 音声プレイヤー
8. 本文
9. Sources

## 対応方針

利用者指定の並びに変更する:

1. パンくずリンク
2. イラスト
3. タイトル
4. レベル・長さバッジ＋ Author / Created
5. 本文

利用者が言及していない要素は次の位置に置く:

- バージョンリンク: パンくず直後（ナビゲーション要素としてまとめる）
- 音声プレイヤー: メタ情報の後・本文の前（リスニング用のため本文直前が自然）
- Sources: 従来どおり本文の後

## 影響範囲

- `lib/site.js` の `renderArticleContent()` のみ（`server.js`・`scripts/build-static-site.js` は共通モジュール経由のため変更不要）
- バリアント詳細ページと記事バージョンページの両方に反映される

## テスト方針

- `node` でトマト記事ページのHTMLをレンダリングし、要素の出現順が指定どおりであることを確認する
- `npm run build` が正常終了することを確認する
