# workflows/add-ideas.md — トピックアイデアの追加

[docs/topic-ideas.md](../docs/topic-ideas.md)（トピック案のストック）に新しいアイデアを追加する単体ワークフロー。
テキスト生成パイプライン（config → … → illustrate）とは独立しており、いつでも実行できる。
新規トピック作成のついでに補充する場合は [workflows/config.md](config.md) 手順7で行う（内容はこのワークフローの手順3〜5と同じ）。

参照: [docs/specs/esl-level-spec.md](../docs/specs/esl-level-spec.md)（ジャンル名・適したレベル帯の定義）

## 手順

### 1. 要望の確認

利用者に以下を確認する（「おまかせ」でもよい）。

- 追加したいカテゴリやテーマの希望（例: 「Science & Nature に」「季節の行事っぽいものを」）
- 件数の希望（未指定なら3〜5件）
- 対象レベル帯の希望（例: 「初級A1〜A2向けを中心に」）

### 2. 現状把握

[docs/topic-ideas.md](../docs/topic-ideas.md) を開き、次を確認する。

- 「アイデア一覧」ツリーの既存カテゴリと在庫の偏り（在庫が薄いカテゴリ・ジャンルはどこか）
- 「採用済み」の一覧（同じ・非常に近いトピックを提案しないため）

### 3. 候補の提案

- 手順1の要望と手順2の在庫状況を踏まえて、候補を考えて利用者に提示する
- 各候補は トピック（英語）/ ジャンル / 適したレベル帯 / メモ の形式で示す
  - ジャンルと適したレベル帯は [esl-level-spec.md](../docs/specs/esl-level-spec.md) の「ESL読解に適した文章形式」表と整合させる
  - メモには「何の練習に向くか」「フィクション扱いにできるか」など後で選ぶときの判断材料を書く
- 利用者に採用する候補を選んでもらう（全部でもよい。入れ替え・修正の要望があれば反映して再提示する）

### 4. ツリーへの追加

- 承認された案を「アイデア一覧」ツリーの適切なカテゴリに追加する。合うカテゴリが無ければ新設してよい（ネストの深さは自由）
- 記法は [docs/topic-ideas.md](../docs/topic-ideas.md) 冒頭の書式説明に従う
  - カテゴリ: `- {カテゴリ名（英語）}`（パイプ `|` を含めない。インデントは2スペース単位）
  - トピック案（葉）: `- {トピック（英語）} | {ジャンル} | {適したレベル帯} | {メモ}`

### 5. 結果の確認

- 追加した内容（カテゴリと件数）を要約して利用者に提示する
- Webビューアの `/topic-ideas`（トップページの「Topic Ideas」リンク）で確認できることを案内する
