# トピックのアイデアページの作成

## 目的・背景

新しいトピックのテキストを作るたびに、その場でトピックを考えるのではなく、あらかじめストックしたアイデアから選べるようにしたい。
また、新規トピック作成のタイミングでアイデアの在庫を補充し、採用済みのトピックがどれかわかるようにして重複作成を防ぐ。
アイデアの在庫と採用状況は閲覧サイト（Webビューア）からも確認できるようにする。

## 対応方針

### Phase 1: アイデアページの作成（実施済み）

1. `docs/topic-ideas.md` を新規作成する
   - 「アイデア一覧」: トピック案（英語タイトル）・ジャンル・メモの表。ジャンルは `esl-level-spec.md` の英名を使う
   - 「採用済み」: 採用したトピックと `texts/` 配下のディレクトリ・採用日を記録する表。既存3トピックを初期データとして記載する
   - 初期アイデアとして各ジャンルからバランスよく候補を用意する
2. `workflows/config.md` を更新する
   - 手順2（トピックの確認）: トピック未定・提案依頼の場合はアイデアページから候補を提示する。トピック指定がある場合も採用済み一覧と照合して重複を防ぐ
   - `config.json` 保存後: 採用したトピックをアイデア一覧から「採用済み」へ移し、新しいアイデアを2〜3件補充する
3. `CLAUDE.md` のディレクトリ構成・説明にアイデアページを追記する

### Phase 2: Webビューアへの表示（フィードバック反映）

1. `lib/site.js` に `renderTopicIdeas(basePath)` を追加する
   - `docs/topic-ideas.md` の「アイデア一覧」「採用済み」各セクションのMarkdownテーブルをパースして表示する（`renderLevels` と同じ方式）
   - 採用済みトピックは `texts/` ディレクトリ名からサイト内のトピックページへリンクする
   - ワークフロー向けの運用ルール（参照・採用・補充の説明）はサイト訪問者には不要なので表示しない
2. `renderIndex` のトップページに「Topic Ideas」へのリンクを追加する（About CEFR Levels の隣）
3. `server.js` に `/topic-ideas` ルートを追加する
4. `scripts/build-static-site.js` で `topic-ideas/index.html` を出力する
5. `docs/topic-ideas.md` に「表の構造を変えるときは `lib/site.js` の `renderTopicIdeas` に影響する」旨の注記を追加する

### Phase 3: アイデア一覧のツリー構造化（フィードバック反映）

アイデアをカテゴリで分類できるよう、「アイデア一覧」をフラットな表からネストした箇条書きのツリーに変更する。

1. `docs/topic-ideas.md` の「アイデア一覧」をツリー形式に書き換える
   - カテゴリノード: `- {カテゴリ名}`（パイプ無し）。任意の深さでネストできる
   - 葉（トピック案）: `- {トピック（英語）} | {ジャンル} | {適したレベル帯} | {メモ}`
   - 既存15案を Everyday Life / Work & School / Travel & Places などのカテゴリに振り分ける
2. `lib/site.js` の `renderTopicIdeas` をツリーパース＋ネストした `<ul>` 表示に変更する（「採用済み」は表のまま）
3. `workflows/config.md` の参照・採用・補充の記述をツリー前提の表現に更新する
   （補充時は適切なカテゴリに追加し、合うカテゴリが無ければ新設する）

### Phase 4: アイデア追加の単体ワークフロー（フィードバック反映）

アイデアの補充が config.md 手順7（新規トピック作成時のついで）にしか無いため、
アイデア追加だけを単体で実行できるワークフロー `workflows/add-ideas.md` を追加する。

1. `workflows/add-ideas.md` を新規作成する（要望の確認 → 現状把握 → 候補の提案 → ツリーへの追加 → 結果の確認）
2. `CLAUDE.md` のディレクトリ構成・ワークフロー一覧に追記する（生成パイプラインとは独立の単体ワークフローとして）
3. `workflows/config.md` 手順7と `docs/topic-ideas.md` から相互参照する

## 影響範囲

- 新規作成: `docs/topic-ideas.md`
- 変更: `workflows/config.md`, `CLAUDE.md`, `lib/site.js`, `server.js`, `scripts/build-static-site.js`
- 生成済みテキスト（`texts/` 配下）には影響しない

## テスト方針

- `docs/topic-ideas.md` の採用済み一覧が `texts/` 配下の実際のトピックと一致していること
- `workflows/config.md` の手順の流れ（参照→採用→補充）が矛盾なく読めること
- ローカルサーバ（`npm start`）で `/topic-ideas` が表示され、トップからのリンク・採用済みトピックへのリンクが機能すること
- `npm run build` で `dist/topic-ideas/index.html` が生成され、basePath 付きのリンクになっていること
