# 記事ページに読み上げキャラ名を表示する

## 目的・背景

記事ページの音声プレイヤーには、どのキャラ（声）が読んでいるかの表示がない。
リスナーが声の主を認識できるよう、読み上げキャラ名をプレイヤーの直前に表示する。
表示名は利用者指定により chobi → **Chobi**、manabi → **Naruko**（なるこ。manabi は旧名）とする。

## 対応方針

- `lib/site.js` の `audioBlock()` で、音声メタデータ `audio/v{N}.json` の `character` を読み、
  表示名マップ（chobi → Chobi / manabi → Naruko、未知の値はそのまま表示）で
  「Read by {Name}」ラベルをプレイヤーの直前に出す
- メタデータが無い・壊れている場合はラベル無しでプレイヤーのみ表示（従来どおり）
- CSS: `.audio-reader` を creator-meta と同系の控えめなスタイルで追加。
  印刷時はプレイヤー（`audio.listen`）と同様に非表示にする

## 影響範囲

- `lib/site.js` のみ（`audioBlock()`・スタイル）。バリアント詳細ページ・記事バージョンページの両方に反映

## テスト方針

- トマトの B1-normal ページをレンダリングし、「Read by Chobi」がプレイヤー直前に出ることを確認
- メタデータJSONが無いケース（存在しないバージョン相当）でラベルが出ず壊れないことを確認
- `npm run build` の正常終了を確認
