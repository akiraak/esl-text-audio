# 印刷時のデザイン修正（音声プレーヤー表示・バッジ枠線）

## 目的・背景

記事ページを印刷したとき、現状の印刷CSSでは以下の不満がある。

1. 音声プレーヤー（`audio.listen`）が `display: none` で完全に消える。
   紙面でも「この記事にはリスニング音声がある」ことが分かるよう、シークバーのデザインは印刷にも表示したい。
2. レベル・分量tierのバッジ（`.badge-level` / `.badge-tier`）は印刷時に背景が白へ強制されるため、
   文字だけが浮いてバッジと分からない。ふち枠線を付けて判別できるようにする。

## 対応方針

`lib/site.js` の `@media print` ブロックのみ変更する（画面表示は変えない）。

1. 非表示指定 `audio.listen, .audio-reader { display: none; }` を削除し、
   音声プレーヤーと "Read by ..." 行を印刷にも表示する
   - **追記**: ネイティブの audio コントロールはバー部分が背景扱いで印刷されず時間表示だけ残るため、
     印刷時は audio 要素を隠し、枠線・文字だけで描いたプレーヤー風ブロック（`.audio-print`、
     ▶アイコン＋トラック線＋つまみ＋メタデータの `durationSeconds` による再生時間表示）に差し替える方式へ変更した
2. `.badge` に `border: 1px solid #999;` を追加する（印刷時の他の枠線色 #999 に合わせる）。
   レベル・tierに限らず一覧ページのジャンル・レベル帯バッジも同じ扱いで統一する

## 影響範囲

- `lib/site.js`（`layout()` 内の印刷CSSのみ）。`server.js`・`scripts/build-static-site.js` は共通モジュール経由のため個別変更不要

## テスト方針

- [x] `npm start` のローカルサーバで記事ページを開き、印刷プレビューで
  音声プレーヤーが表示されること・バッジに枠線が付くことを確認
  （headless Chrome の print-to-pdf で `water-cycle` B1-normal 記事ページを検証。
  バッジ枠線・シークバー表示・"Read by ..." の純黒化を確認）
- [x] `npm run build` がエラーなく完走することを確認
